10-Week Development Plan

🎯 Project Objectives

▾
  • Build a fully functional campus waste management and asset reporting web application within 10 weeks.
  • Deliver role-based dashboards for Students, Teachers, Admin, and MRF Staff with real-time report tracking.
  • Implement a secure authentication system with JWT-based session management and role-based access control.
  • Design and deploy a PostgreSQL database with Prisma ORM supporting Users, Reports, and Audit Logs.
  • Achieve a complete report lifecycle workflow: PENDING → VERIFIED → DISPATCHED → COLLECTED → RESOLVED.
  • Create responsive, mobile-first UI using React 19, TailwindCSS 4, and Framer Motion animations.

✅ Success Criteria

▾
  • Timely Completion: All features delivered by Week 10 with no critical blockers remaining.
  • Functional Quality: All 4 role dashboards fully operational with complete CRUD operations.
  • Authentication: Secure sign-in/sign-out with proper JWT token validation and role-based routing.
  • Data Integrity: Report lifecycle transitions are validated — no invalid state changes allowed.
  • Responsive Design: Application works on desktop (≥1024px), tablet (768px), and mobile (≤480px).
  • Budget Compliance: Total expenses stay within fundraised budget; no financial losses.
  • Documentation: Complete technical docs, user guide, and project report submitted on time.
